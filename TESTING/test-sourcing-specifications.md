# Test Sourcing Specifications for RFP Design Agent

This document contains realistic sourcing specifications for testing the RFP Design Agent's ability to create well-structured RFPs.

---

## 1. Simple Product Procurement - Office Supplies

**Specification:**
```
We need to purchase office supplies for our team of 50 employees:
- Pens (ballpoint, blue and black ink)
- Notebooks (spiral bound, college ruled)
- Sticky notes (3x3 inch, various colors)
- File folders (letter size, manila)
- Stapler and staples

Estimated monthly usage:
- 100 pens
- 50 notebooks  
- 200 sticky note pads
- 100 file folders
- 5 boxes of staples

Budget: $500/month
Delivery: Monthly deliveries to our office in Seattle, WA
Contract term: 12 months with option to renew
```

**Expected Agent Actions:**
- Create basic product specifications with quantities
- Set up simple evaluation criteria (price, quality, delivery reliability)
- Define straightforward contract terms
- Create a simple scoring matrix

## Testing Scenarios

### Basic Test Flow:
1. User provides specification to RFP Design Agent
2. Agent should:
   - Acknowledge the sourcing need
   - Ask clarifying questions if needed
   - Create structured RFP with all sections
   - Suggest evaluation criteria appropriate to the procurement type
   - Recommend timeline for the RFP process
   - Create initial draft for user review

### Advanced Test Scenarios:

**Scenario A: Iterative Refinement**
- User provides basic spec (Example 1)
- Agent creates initial RFP
- User requests additions: "Add sustainability requirements"
- Agent updates RFP appropriately

**Scenario B: Multi-Agent Coordination**
- User provides complex spec (Example 5)
- RFP Design Agent creates technical RFP
- User switches to Sourcing Agent to discuss vendor outreach
- Agents maintain context about the same procurement

**Scenario C: Compliance Focus**
- User provides healthcare-related spec
- Agent should automatically include HIPAA, security requirements
- Agent should recommend additional compliance documentation

**Scenario D: Budget Optimization**
- User provides spec with budget constraints
- Agent should recommend evaluation criteria that balance cost vs. value
- Agent should suggest optional vs. required features to manage costs

## Expected Agent Capabilities

The RFP Design Agent should demonstrate:

1. **Context Understanding**: Recognize procurement type and adjust RFP structure accordingly
2. **Requirements Extraction**: Parse unstructured specifications into organized sections
3. **Best Practices**: Apply industry-standard RFP practices for the procurement category
4. **Compliance Knowledge**: Identify regulatory requirements based on industry and goods/services
5. **Evaluation Framework**: Create appropriate weighted evaluation criteria
6. **Timeline Estimation**: Suggest realistic timelines based on procurement complexity
7. **Risk Identification**: Flag potential issues or missing information
8. **Professional Output**: Generate well-formatted, professional RFP documents

---

## Success Metrics

For each test specification, the agent should:
- ✅ Create complete RFP with all standard sections
- ✅ Extract and organize all requirements from the specification
- ✅ Add appropriate evaluation criteria
- ✅ Identify any missing critical information
- ✅ Format output professionally
- ✅ Complete task within reasonable conversation flow (5-10 exchanges max)
- ✅ Maintain context if user makes iterative changes
